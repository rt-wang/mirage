What This App Is 02 / 09

The core idea is an object-first visual instrument.

Detection finds attention COCO-SSD identifies objects in the mirrored webcam frame.
Local CV finds structure OpenCV runs edges and lines inside each detected object box.
Actions create style The renderer composes deterministic visual effects from an ActionPlan.

Speaker beat: This is no longer a multi-mode CV demo. The object detector is the root layer, and every creative effect hangs off tracked object regions.

Talk time: 35 seconds Design thesis
Runtime Pipeline 03 / 09

The app is one loop with clear handoffs.

Camera getUserMedia creates the hidden video source.
Capture app.js mirrors each frame into captureCanvas.
Detect objectDetector.js runs COCO-SSD.
Track objectTracker.js assigns stable IDs.
Local CV objectLocalCv.js extracts object-local geometry.
Render actionRenderer.js draws the active plan.

Speaker beat: The important architectural move is that every stage returns structured data to the next stage. The renderer is not re-detecting objects or rethinking analysis.

Talk time: 45 seconds One frame at a time

Data Model 05 / 09

Everything depends on the tracked object as the shared contract.

DetectedObject Stable ID, class name, confidence, smoothed bbox, center, area, velocity, selected flag, and stale state.
ObjectGeometry Local edge mask, local Hough lines, geometry origin, bbox, and future local-depth or freeze data.
Object detection decides where the system looks. Object-local CV decides what structure is available there. Action plans decide how it feels.

Speaker beat: This keeps the LLM layer clean later. It does not need raw pixels. It only needs object classes, scene signals, and the action schema.

Talk time: 40 seconds Shared contracts
Rendering System 06 / 09

The renderer is a small compositor, not a pile of presets.

Global style Source opacity, tint, contrast, saturation, grain, and blend mode.
Object rules Selectors choose all objects, specific classes, largest object, or selected object.
Actions Aura, trail, spotlight, glitch, local edges, and local lines compose per object.

Speaker beat: The four visible presets are not the limit. They are sample JSON plans. The renderer can accept many new combinations as long as they validate against the safe action vocabulary.

Talk time: 40 seconds ActionPlan mental model
Current Actions 07 / 09

The creative vocabulary is already modular.

localEdges Recolors Canny masks inside object boxes with glow and thickness.
localLines Draws Hough segments with line weight, color, and jitter.
aura Creates a pulsing radial field centered on an object.
trail Uses a persistent canvas to smear object crops over time.
spotlight Dims the world outside an object-centered region.
glitch Slices and displaces object crops over the styled source.
Talk time: 35 seconds Vocabulary, not modes
Phase 4 08 / 09

The next layer is an LLM planner that chooses safe JSON.

What the LLM can change Object targets, labels, colors, blend mode, global style, and combinations of supported actions.
What the LLM cannot change JavaScript, OpenCV calls, model loading, network URLs, arbitrary CSS, or unsupported action names.
Phase 4 should add validation first, then prompt UI, then richer actions like localDepth and freezeBox.

Speaker beat: This gives AI a real creative surface without making the browser execute model-generated code. The model is an art director, not a runtime.

Talk time: 45 seconds Safe expansion path
Closing Summary 09 / 09

How to explain the whole codebase in one sentence:

Latent Canvas captures webcam frames, detects and tracks objects, extracts local geometry from each object box, then renders deterministic creative actions selected by presets today and by a validated LLM planner next.

Stable center The tracked object is the main abstraction.
Clean boundary Analysis creates geometry. Rendering consumes geometry.
Future-ready The ActionPlan schema is the bridge to AI direction.
